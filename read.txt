## 1. LOGIKA MATEMATIS, MATA UANG & RISK MANAGEMENT
1. Mata Uang & Batasan: Menggunakan mata uang IDR (Rupiah). AI yang menentukan besaran taruhan secara dinamis, namun WAJIB berada di rentang minimal Rp2.000 (2000.0) dan maksimal Rp5.000 (5000.0) per taruhan.
2. Value Bet Detector: Edge = Probabilitas AI - Probabilitas Implisit Bandar (1 / Odds). Taruhan hanya valid jika Edge >= MIN_VALUE_EDGE.
3. Size Taruhan (AI-Driven): Gunakan Fractional Kelly Criterion:
   - f = ((b * p) - q) / b  | di mana b = Odds - 1, p = Probabilitas AI, q = 1 - p.
   - Hasil nominal = f * KELLY_MULTIPLIER * CURRENT_BANKROLL.
   - AI wajib melakukan kliping (clipping): Jika nominal < 2000, batalkan bet (return 0). Jika nominal > 5000, potong menjadi 5000.
4. Proteksi Drawdown: Jika akumulasi `daily_loss` >= MAX_DAILY_DRAWDOWN * BANKROLL, bot wajib menghentikan taruhan baru untuk hari itu (return stake = 0).

## 3. SPESIFIKASI KODE PER FILE (MOHON TULIS SECARA UTUH)

### A. config.py
Tulis parameter berikut dengan Type Hinting:
- INITIAL_BANKROLL: float = 100000.0  # Contoh saldo awal Rp100.000
- KELLY_MULTIPLIER: float = 0.10      # Multiplier konservatif untuk menjaga stabilitas
- MIN_VALUE_EDGE: float = 0.05        # Minimal edge 5%
- MIN_BET_IDR: float = 2000.0         # Hard limit minimum Rp2.000
- MAX_BET_IDR: float = 5000.0         # Hard limit maksimum Rp5.000
- MAX_DAILY_DRAWDOWN: float = 0.10     # Maksimal rugi 10% per hari dari bankroll
- STAKE_API_URL: str = "https://api.stake.com/graphql"
- STAKE_SESSION_TOKEN: str = "GANTI_DENGAN_COOKIE_X_ACCESS_TOKEN_ANDA"
- STAKE_CURRENCY: str = "idr"
- SIMULATION_MODE: bool = True        # Tetap berikan opsi ini untuk testing awal

### B. predictor.py
- Fungsi `get_ensemble_prediction(match_data: dict) -> dict`: Mengembalikan probabilitas 3-Way `{"Home": float, "Draw": float, "Away": float}` (Total wajib di-normalisasi menjadi 1.0).
- Fungsi `get_multi_agent_consensus(match_data: dict) -> dict`: Simulasikan diskusi 3 Agen LLM (Statistisi, Taktis, Sentimen). Hasil diskusi wajib diekstrak menjadi nilai float numerik untuk probabilitas akhir match tersebut.

### C. arbitrage_finder.py
- Fungsi `scan_opportunities(odds_data: dict, ai_probs: dict) -> list`:
  - Menerima input data odds multi-bookmaker (termasuk pasar Stake) dalam format Odds Desimal.
  - Scan celah Arbitrase dan Value Bet. Kembalikan list of dict berisi detail peluang, odds terbaik, target outcome, dan platform tempat pasang taruhan.

### D. bet_sizer.py
- Fungsi `calculate_idr_stake(odds: float, prob: float, current_bankroll: float, current_daily_loss: float) -> float`:
  - Hitung nominal taruhan menggunakan rumus Fractional Kelly.
  - Terapkan logika pengetatan: jika nominal di luar rentang `MIN_BET_IDR` dan `MAX_BET_IDR`, sesuaikan nilainya secara otomatis (atau kembalikan 0 jika terlalu berisiko).
  - Validasi batas drawdown harian. Jika terlampaui, kembalikan `0.0`.

### E. executor.py
- Buat class `StakeExecutor`. Jika `SIMULATION_MODE = False`, implementasikan fungsi `place_stake_bet(active_odds_id: str, target_stake: float)` menggunakan library `requests` untuk menembak GraphQL API Stake dengan mutasi `createSportsBet`.
- Wajib menyertakan header `"x-access-token"` yang diisi dari `config.STAKE_SESSION_TOKEN`.
- Simpan setiap riwayat taruhan dan statusnya ke file `bet_history.json`.

### F. main.py
- Buat orchestrator utama yang berjalan dalam `while True` loop setiap 10 menit.
- Bungkus alur data dengan `try-except Exception as e` yang kuat agar bot tidak mati total jika terjadi gangguan jaringan (API timeout) atau kegagalan autentikasi token.
